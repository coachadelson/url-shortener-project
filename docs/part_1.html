Part 1: Building a Simple URL Shortener

Summary

Welcome to the first step of our expansive project journey: Building a URL Shortener. Behind the simplicity of a shortened URL lies a myriad of technological components and design principles. Take a look at Bitly, a service similar to what we're trying to build, albeit ours is much simpler. Play with it and imagine how one would go about building it. Even when relegating your mental models to DSA concepts, you should be able to come up with something with hashmaps/dictionaries. With this exercise, we will learn how to transition these ideas to a system at large.

Objectives

In this initial phase, you'll establish a foundational understanding by constructing a basic URL shortener using FastAPI. This will not only introduce you to the nuances of web application development but will also serve as a gateway to mastering a diverse set of tools and methodologies.

Key Learning Points

Throughout the project's progression, you will:

  • Engage with Key Technologies: Delve deep into various platforms and libraries like GitHub, FastAPI, DynamoDB, Postman, and more. These tools, while distinct in function, collectively empower you to design, develop, test, and deploy robust applications.

  • Adopt Best Design Practices: Learn the art and science behind database design, API endpoint construction, and application architecture. Crafting a resilient and efficient system isn't just about code, it's also about design.

  • Embrace Testing: Understand the importance of testing in software development. From unit tests that validate individual components to end-to-end tests that ensure the system's holistic functionality, you'll learn to ensure your application's reliability and robustness.

  • Collaborate and Review: Dive into the collaborative nature of development using platforms like GitHub. Understand the lifecycle of code changes, from branching and pull requests to code reviews and conflict resolution.

Exit Criteria

Here's a checklist that specifies the expected outcomes or behaviors to be demonstrated.

Loom "End to End Test" Submission Checklist:

  1. Introduction:

  2. Briefly explain the purpose of the video.

  3. Provide a short overview of the features developed.

  4. API Endpoints Demonstration:

  5. POST /shorten_url:

  6. Showcase shortening a long URL without providing a custom short_url.

  7. Showcase shortening a long URL with a custom short_url.

  8. Demonstrate how the system responds when trying to use a custom short_url that's already been taken.

  9. GET /list_urls:

  10. Show the list of all shortened URLs and their corresponding original URLs.

  11. GET /redirect :

  12. Use a shortened URL to showcase successful redirection to the original long URL.

  13. Use an invalid or non-existent shortened URL to demonstrate the error handling and response.

  14. Error Handling:

  15. Input an invalid URL format to the POST /shorten_url endpoint and demonstrate the error response.

  16. Try shortening the same URL with the same custom short_url twice to see how the system handles duplicates.

  17. Database Interactions:

  18. Briefly showcase the database structure (if possible). Explain how URLs are stored and how the short URL corresponds to the long URL.

  19. Mention how unique short URLs are ensured.

  20. Postman Demonstrations:

  21. Showcase the actual HTTP requests and responses using Postman. This gives a clear picture of the API's behavior from a technical standpoint.

  22. Browser Redirection:

  23. Copy-paste a shortened URL into a web browser and show that it successfully redirects to the original long URL.

  24. Mention the speed of redirection and any other notable features or observations.

  25. Closing Remarks:

  26. Highlight any challenges faced and how they were overcome.

  27. Briefly discuss potential improvements or features that could be added in the future.
