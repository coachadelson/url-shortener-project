Part 2: Deploying and Hosting

Summary

Now that you have built a functional URL shortener, the next step is to make it accessible to the world. In this phase, you will venture into the world of cloud computing and web hosting, taking your local application and deploying it to a cloud provider. By the end, your URL shortener will be live on the internet, ready to shorten URLs for anyone who accesses it.

Objectives:

  • Understand cloud hosting and infrastructure.

  • Engage with deployment strategies.

  • Set up a Continuous Integration and Continuous Deployment (CI/CD) pipeline.

  • Host the application on a cloud platform and ensure its stability and scalability.

Your Task: Deploy to AWS and Set Up a CI/CD Pipeline

Setting Up Your AWS Account

  1. Sign Up for AWS:

    • If you haven’t already, sign up for an AWS account.

    • Familiarize yourself with the AWS Management Console. Look into EC2 (Elastic Compute Cloud) and S3 (Simple Storage Service).

    • Understand AWS pricing, especially for EC2 and S3, so that you can stay within the free tier.

Deployment with Docker

  1. Containerize Your Application:

    • This will make deployments smoother and more consistent.

    • Create a Dockerfile in the root directory.

    • Use appropriate base images for Python and FastAPI.

    • Ensure all dependencies are installed.

    • Build your Docker image and test it locally to ensure your application runs as expected.

Deploying to AWS EC2

  1. Launch an EC2 Instance:

    • Go to the AWS Management Console and launch a new EC2 instance.

    • Choose an appropriate instance type (t2.micro is a good start if you're within the free tier).

    • Configure security groups to allow HTTP (port 80) and HTTPS (port 443) traffic.

  2. SSH into Your EC2 Instance:

    • Use an SSH client to connect to your EC2 instance.

  3. Set Up Your EC2 Instance:

    • Install Docker on your EC2 instance.

    • Pull your Docker image from Docker Hub or AWS ECR.

    • Run your containerized application on EC2.

    • Ensure your application is accessible via the EC2 instance's public IP address

Setting Up a CI/CD Pipeline with GitHub Actions

  1. Understand GitHub Actions Basics:

    • GitHub Actions allows you to automate workflows directly in your GitHub repository.

  2. Set Up a Workflow:

    • Create a .github/workflows/main.yml file in your repository.

    • Configure the workflow to:

      • Check out your code.

      • Build your Docker image.

      • Push the Docker image to Docker Hub or AWS ECR.

      • Deploy the Docker image to your EC2 instance automatically whenever there's a merge to the main branch.

  3. Handle Secrets Securely:

    • Use GitHub Secrets to store sensitive information like AWS credentials.

Domain Name and HTTPS

  1. Setup a Domain Name:

    • Purchase a domain name from a domain registrar if you don't already have one.

    • Use AWS Route 53 to manage your domain.

  2. Configure AWS Route 53:

    • Point your domain to your EC2 instance's public IP address.

  3. Set Up HTTPS:

    • Use Let's Encrypt to obtain a free SSL certificate.

    • Configure your EC2 instance to use the SSL certificate for HTTPS traffic.

Scaling and Load Balancing (Optional)

  1. Understand Load Balancers:

    • Load balancers distribute incoming traffic to multiple EC2 instances to ensure high availability and reliability.

  2. Set Up an AWS Load Balancer:

    • Configure an AWS Load Balancer to distribute traffic across multiple EC2 instances.

    • Ensure horizontal scalability by launching more EC2 instances if required.

Steps

  1. AWS Setup and Familiarization

    • Sign up for AWS.

    • Navigate the AWS Management Console.

    • Review AWS pricing details.

  2. Dockerize the Application

    • Write a Dockerfile.

    • Build and test the Docker image locally.

  3. Deploy to EC2

    • Launch and set up an EC2 instance.

    • Deploy your Docker container on EC2.

    • Verify the application is running and accessible.

  4. CI/CD with GitHub Actions

    • Write a GitHub Action workflow for CI/CD.

    • Securely handle secrets.

    • Test the CI/CD pipeline by making a change and observing automatic deployment.

  5. Domain and HTTPS

    • Set up a domain.

    • Configure Route 53.

    • Set up HTTPS using Let's Encrypt.

  6. Scale and Load Balance

    • Understand the concept of load balancers.

    • Deploy an AWS Load Balancer.

    • Test horizontal scalability.

Technologies You Will Learn to Use

  • AWS: The leading cloud service provider. You'll primarily use EC2, S3, and Route 53.

  • Docker: A tool to create, deploy, and run applications in containers.

  • GitHub Actions: Automate workflows directly in your GitHub repository.

  • Let's Encrypt: A free, automated, and open Certificate Authority.

Exit Criteria

Upon completion of this phase, you should:

  • Have your URL shortener live on the internet, accessible via a custom domain name.

  • Ensure the application is running over HTTPS.

  • Have a functioning CI/CD pipeline that automatically deploys changes from your GitHub repository.

  • (Optional) Demonstrate that your application can handle traffic spikes by scaling and distributing traffic.
