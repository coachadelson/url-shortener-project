Part 3: Users and Permissions

Summary

Building on the foundational URL shortener from Part 1 and the deployment steps from Part 2, we now introduce the concepts of user management, permissions, and security. The overarching goal of this phase is to evolve our application from a generic URL shortener to a more personalized tool where users have individual accounts with specific permissions and associated limitations. By the end of this module, you'll grasp how to handle users, authenticate their sessions, manage their permissions, and ensure data security.

Objectives

  • Implement user authentication and management.

  • Establish permissions and access control.

  • Ensure secure password handling.

  • Build and test new API endpoints for user management

Key Concepts

JSON Web Tokens (JWT) for Authentication: JWT is a compact and self-contained way for securely transmitting information between parties as a JSON object. They can be signed using a secret or a public/private key. For our purposes, they'll be essential for session management and ensuring that users are who they say they are.

Password Security: In the digital age, protecting users' sensitive data, especially passwords, is paramount. You can very easily lose your user's trust if their sensitive information is easily accessible or retrievable during an attack. By understanding hashing techniques, you'll ensure passwords are stored securely and are resistant to malicious attacks.

FastAPI Authentication Decorators: FastAPI provides robust tools and decorators that handle user authentication and permissions. By harnessing these tools, you'll ensure only authorized users can access specific resources and perform allowed operations.

Your Task: User Management and Permissions

APIs to Implement:

  1. /create_user: This endpoint creates a new user with a username and password. By default, the user can create a maximum of 20 URLs.

xxxxxxxxxx
 
POST /create_user
Body:
{
    "username": "newUser",
    "password": "strongPassword123"
}
​
Response 200:
{
    "message": "User 'newUser' created successfully."
}


  1. /list_urls: Update this endpoint to allow only admin users to view all the URLs.


  1. /list_my_urls: This endpoint allows a user to list their shortened URLs.

xxxxxxxxxx
 
GET /list_my_urls
​
Response 200 (for a user with two URLs):
[
    {
        "short_url": "userabc1",
        "original_url": "https://www.exampleA.com"
    },
    {
        "short_url": "userabc2",
        "original_url": "https://www.exampleB.com"
    }
]


  1. /change_password: Allow users to change their password.

xxxxxxxxxx
 
POST /change_password
Body:
{
    "old_password": "strongPassword123",
    "new_password": "evenStrongerPassword456"
}
​
Response 200:
{
    "message": "Password changed successfully."
}


  1. /create_url: Limit the number of URLs a user can create based on their set limit. Admin users can adjust this limit.


  1. /update_url_limit: Admin users can change the URL creation limit for other users.

xxxxxxxxxx
 
POST /update_url_limit
Body:
{
    "username": "someUser",
    "new_limit": 25
}
​
Response 200:
{
    "message": "Limit updated successfully for 'someUser'."
}


Ensure that errors are thrown if an admin tries to set a limit lower than the current number of URLs a user has.

Steps:

  1. Setup JWT Authentication:

    • Integrate JWT into your FastAPI server to manage sessions and user authentication.

    • Recommended Reading: Introduction to JSON Web Tokens.

  2. Password Hashing:

    • Ensure all user passwords are hashed before being stored in the database. You can use libraries like Bcrypt or Argon2 for this purpose.

    • Recommended Libraries: Bcrypt, Argon2.

  3. Implement User Management APIs:

    • Extend your FastAPI server to handle the new endpoints. Integrate them with your database to store and retrieve user-related data.

    • Tutorial: FastAPI - Security and Authentication.

  4. Admin Privileges:

    • Ensure that certain endpoints, such as /list_urls and /update_url_limit, are only accessible by users with admin privileges.

  5. Testing:

    • Just like in Part 1, create a testing plan and use Postman to ensure that your endpoints are working correctly.

    • Pay special attention to edge cases, such as non-admin users trying to access admin-only endpoints, or users trying to exceed their URL limit.

Exit Criteria:

Your task will be deemed complete when you can demonstrate the following:

  • Successfully register a new user.

  • Authenticate as a user and create short URLs up to their limit.

  • Change a user's password.

  • Authenticate as an admin and adjust a user's URL limit.

  • Handle errors correctly, such as non-admin users trying to access admin-only endpoints or users trying to exceed their URL limit.
